# Vendored play engine (from HimkageVision)

This directory is a **vendored copy** of the HimkageVision play-diagram engine, used by the
Playcall game (`src/playcall/`) to resolve real matchups and render live diagrams. It is the
canonical, test-verified football geometry source (see `../../FOOTBALL-KNOWLEDGE-CANON.md`).

It is plain CommonJS with no build step and is **required at runtime from the repo root** (not
compiled into `dist/`), the same way `data/` and `content/` are read, so it survives `tsc`.

## Files
- `play-engine.core.js` — the resolver + SVG renderer. Public API: `resolvePlay`, `resolvePass`,
  `resolveShell`, `classifyRun`, `classifyPass`, `renderSVG`, `buildModel`, `positionsAt`.
- `play-data.js` — the corpus (43 formations, 49 fronts, 7 coverages, 12 run schemes, 26 pass
  concepts, 455 plays), generated by HimkageVision's `build-data.js` from its `data/*.json`.
- `fonts/PlayartSans*.ttf` — Barlow (SIL Open Font License), bundled so headless hosts rasterize
  diagram text deterministically (`@resvg/resvg-js` with `loadSystemFonts:false`).

## Re-syncing after a HimkageVision change
1. In `C:\iMoveChainz\HimkageVision`: edit `data/*.json`, then `node build-data.js`.
2. Copy `play-engine.core.js` + `play-data.js` here.
3. Run `npm run smoke` and `npm run sim:playcall` to confirm nothing regressed.
